What is a Server?

A server is a computer program or device that provides services to other programs or devices, called clients. Understanding servers is fundamental to backend development.

Server Definition

In the context of web development, a server is a program that runs continuously, listening for requests from clients (usually web browsers) and responding with the requested data, files, or services.

Database Server

Manages data storage, retrieval, and database operations.

API Server

Provides programmatic access to services via APIs.

REST GraphQL

Server Responsibilities

  • Request Handling: Receiving and parsing HTTP requests
  • Route Processing: Determining which code to execute based on the request
  • Business Logic: Processing data and executing application logic
  • Data Management: Interacting with databases and external services
  • Response Generation: Creating and sending HTTP responses
  • Security: Authentication, authorization, and data protection
  • Performance: Optimizing response times and resource usage

Basic Server Example

Here's a simple server that demonstrates core concepts:

const http = require('http');
const url = require('url');

const server = http.createServer((req, res) => {
  const parsedUrl = url.parse(req.url, true);
  const path = parsedUrl.pathname;
  const method = req.method;

  // Set response headers
  res.setHeader('Content-Type', 'application/json');
  res.setHeader('Access-Control-Allow-Origin', '*');

  // Route handling
  if (path === '/' && method === 'GET') {
    res.statusCode = 200;
    res.end(JSON.stringify({ 
      message: 'Welcome to our API',
      timestamp: new Date().toISOString()
    }));
  } else if (path === '/users' && method === 'GET') {
    res.statusCode = 200;
    res.end(JSON.stringify({ 
      users: [
        { id: 1, name: 'John Doe' },
        { id: 2, name: 'Jane Smith' }
      ]
    }));
  } else {
    res.statusCode = 404;
    res.end(JSON.stringify({ 
      error: 'Route not found' 
    }));
  }
});

const PORT = process.env.PORT || 3000;
server.listen(PORT, () => {
  console.log(`Server running on port ${PORT}`);
});

Key Takeaway: This example shows basic request handling, routing, and response generation - the core functions of any server.
