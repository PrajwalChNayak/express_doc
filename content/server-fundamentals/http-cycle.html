HTTP Protocol

The HyperText Transfer Protocol (HTTP) is the foundation of data communication on the World Wide Web. Understanding HTTP is crucial for backend development.

What is HTTP?

HTTP is a stateless, application-layer protocol that defines how messages are formatted and transmitted between web browsers (clients) and web servers. It follows a request-response model.

HTTP Request-Response Cycle

1

Client sends HTTP request

Browser initiates request to server

2

Server processes request

Server parses request and executes logic

3

Server sends HTTP response

Server returns data or error to client

4

Connection closed

HTTP is stateless - connection ends

HTTP Request Structure

An HTTP request consists of four main parts:

POST /api/users HTTP/1.1
Host: example.com
Content-Type: application/json
Authorization: Bearer token123
Content-Length: 45

{
  "name": "John Doe",
  "email": "john@example.com"
}

1. Request Line

Contains the HTTP method, URL path, and HTTP version.

POST /api/users HTTP/1.1

2. Headers

Key-value pairs providing metadata about the request.

Content-Type: application/json

3. Empty Line

Separates headers from the body.

4. Body (Optional)

Contains data sent to the server (for POST, PUT, PATCH).

HTTP Methods

Method Purpose Safe Idempotent
GET Retrieve data Yes Yes
POST Create new resource No No
PUT Update/replace resource No Yes
PATCH Partial update No No*
DELETE Remove resource No Yes

HTTP Response Structure

An HTTP response has a similar structure to requests:

HTTP/1.1 201 Created
Content-Type: application/json
Location: /api/users/123
Content-Length: 78

{
  "id": 123,
  "name": "John Doe",
  "email": "john@example.com",
  "createdAt": "2024-01-15T10:30:00Z"
}

Status Codes

2xx Success

  • 200 - OK
  • 201 - Created
  • 204 - No Content

3xx Redirection

  • 301 - Moved Permanently
  • 302 - Found
  • 304 - Not Modified

4xx Client Error

  • 400 - Bad Request
  • 401 - Unauthorized
  • 404 - Not Found

5xx Server Error

  • 500 - Internal Server Error
  • 502 - Bad Gateway
  • 503 - Service Unavailable

Common Headers

Request Headers

Accept Content types client accepts
Authorization Authentication credentials
User-Agent Client application info
Cookie Session cookies

Response Headers

Content-Type Response data format
Set-Cookie Set cookies on client
Cache-Control Caching directives
Location Redirect URL

HTTP/2 and HTTP/3

Modern HTTP Versions

  • HTTP/2: Multiplexing, server push, header compression
  • HTTP/3: Built on QUIC, reduced latency, better mobile performance
