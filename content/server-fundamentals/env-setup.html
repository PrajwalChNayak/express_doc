Node.js Installation

Node.js is the runtime environment for executing JavaScript on the server. Let's install it using the recommended methods.

Using Node Version Manager (NVM)

NVM allows you to install and switch between multiple Node.js versions.

Install NVM (macOS/Linux) 
# Install NVM
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash

# Reload your terminal or run:
source ~/.bashrc

# Install latest LTS Node.js
nvm install --lts
nvm use --lts
nvm alias default node
NVM Commands 
# List available versions
nvm list-remote

# Install specific version
nvm install 18.17.0

# Use specific version
nvm use 18.17.0

# Set default version
nvm alias default 18.17.0

# Check current version
node --version

Package Managers

Choose a package manager for dependency management. Each has its own benefits.

npm

Default package manager that comes with Node.js

# Install dependencies
npm install

# Add dependency
npm install express

# Global install
npm install -g nodemon

Yarn

Faster alternative with better caching

# Install Yarn
npm install -g yarn

# Install dependencies
yarn install

# Add dependency
yarn add express

pnpm

Efficient package manager with shared storage

# Install pnpm
npm install -g pnpm

# Install dependencies
pnpm install

# Add dependency
pnpm add express

Environment Variables

Environment variables help manage configuration across different environments securely.

.env.example 
# Server Configuration
PORT=3000
NODE_ENV=development

# Database
DATABASE_URL=mongodb://localhost:27017/myapp
DB_NAME=myapp

# Authentication
JWT_SECRET=your-super-secret-jwt-key
JWT_EXPIRES_IN=7d

# External APIs
STRIPE_SECRET_KEY=sk_test_...
SENDGRID_API_KEY=SG...

# File Upload
CLOUDINARY_CLOUD_NAME=your-cloud-name
CLOUDINARY_API_KEY=your-api-key
CLOUDINARY_API_SECRET=your-api-secret
Loading Environment Variables 
// config/env.js
require('dotenv').config();

const config = {
    port: process.env.PORT || 3000,
    nodeEnv: process.env.NODE_ENV || 'development',
    databaseUrl: process.env.DATABASE_URL,
    jwtSecret: process.env.JWT_SECRET,
    jwtExpiresIn: process.env.JWT_EXPIRES_IN || '7d'
};

// Validate required environment variables
const requiredEnvVars = ['DATABASE_URL', 'JWT_SECRET'];
requiredEnvVars.forEach(envVar => {
    if (!process.env[envVar]) {
        console.error(`❌ Missing required environment variable: ${envVar}`);
        process.exit(1);
    }
});

module.exports = config;
