Quick Start Example

Let's start with a simple Express.js application to understand the basics:

Basic Express Server 
const express = require('express');
const app = express();
const PORT = process.env.PORT || 3000;

// Middleware to parse JSON bodies
app.use(express.json());

// Basic route
app.get('/', (req, res) => {
    res.json({ 
        message: 'Welcome to Express.js!',
        timestamp: new Date().toISOString()
    });
});

// API endpoint
app.get('/api/users', (req, res) => {
    res.json([
        { id: 1, name: 'John Doe', email: 'john@example.com' },
        { id: 2, name: 'Jane Smith', email: 'jane@example.com' }
    ]);
});

// POST endpoint
app.post('/api/users', (req, res) => {
    const { name, email } = req.body;
    
    // Basic validation
    if (!name || !email) {
        return res.status(400).json({ 
            error: 'Name and email are required' 
        });
    }
    
    // Simulate creating user
    const newUser = {
        id: Date.now(),
        name,
        email,
        createdAt: new Date().toISOString()
    };
    
    res.status(201).json({
        message: 'User created successfully',
        user: newUser
    });
});

// Error handling middleware
app.use((err, req, res, next) => {
    console.error(err.stack);
    res.status(500).json({ 
        error: 'Something went wrong!' 
    });
});

// 404 handler
app.use('*', (req, res) => {
    res.status(404).json({ 
        error: 'Route not found' 
    });
});

// Start server
app.listen(PORT, () => {
    console.log(`üöÄ Server running on http://localhost:${PORT}`);
});

Key Features That Make Express.js Powerful

üéØ Smart Routing System

Like a GPS for your web requests. When someone visits /users or /products, Express.js automatically knows which code to run. You can even capture dynamic parts like /users/123 to get user ID 123.

app.get('/users/:id', handler);

‚öôÔ∏è Middleware Magic

Think of middleware like security checkpoints at an airport. Each piece of middleware can check, modify, or block requests before they reach your main code. Perfect for authentication, logging, and validation.

app.use(authenticate);

üé® Template Engine Support

Like mail merge for web pages. Create HTML templates with placeholders, and Express.js fills them with real data. Supports popular engines like EJS, Handlebars, and Pug.

app.set('view engine', 'ejs');

üìÅ Static File Serving

Automatically serves images, CSS, JavaScript, and other static files. Just point Express.js to a folder, and it handles the rest - no manual file serving code needed.

app.use(express.static('public'));

Understanding Core Concepts

üîÑ Middleware: The Assembly Line

Imagine a restaurant kitchen where each station handles one task: chopping vegetables, cooking meat, plating food. Middleware works the same way - each piece handles one responsibility like checking if a user is logged in, validating data, or logging requests. The request flows through each middleware in order until it reaches your final route handler.

Flow: Request ‚Üí Authentication ‚Üí Validation ‚Üí Logging ‚Üí Your Code ‚Üí Response

üõ£Ô∏è Routing: The Traffic Director

Routing is like a traffic director at a busy intersection. When a request comes in for GET /users, POST /login, or DELETE /posts/123, the router directs it to the right handler function. You can organize routes by feature (user routes, product routes) and handle different HTTP methods (GET for reading, POST for creating, PUT for updating).

üì® Request & Response: The Conversation

The request object (req) contains everything the client sent: URL parameters, form data, headers, cookies. The response object (res) is your way to reply: sending JSON data, HTML pages, setting cookies, or redirecting users. It's like having a complete conversation with each visitor to your website.

Using Express Application Generator

Express Generator 
# Install Express generator globally
npm install -g express-generator

express --view=ejs my-express-app

# Or create API-only app
express --no-view my-api-app

# Navigate and install dependencies

# Start the application
npm start

Note: The Express generator creates a more opinionated structure with views, public assets, and additional middleware. Great for traditional web applications.
