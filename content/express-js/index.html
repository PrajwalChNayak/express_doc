Express.js Framework Overview

Express.js is the most popular Node.js web application framework, providing a robust set of features for building web applications and APIs. It's minimal, flexible, and provides a solid foundation for building scalable applications.

Why Express.js?

  • Minimal and unopinionated framework
  • Large ecosystem of middleware
  • Excellent performance
  • Great community support
  • Easy to learn and use
  • Flexible routing system
  • Built-in security features
  • Industry standard for Node.js APIs

Quick Start Example

Let's start with a simple Express.js application to understand the basics:

Basic Express Server 
const express = require('express');
const app = express();
const PORT = process.env.PORT || 3000;

// Middleware to parse JSON bodies
app.use(express.json());

// Basic route
app.get('/', (req, res) => {
    res.json({ 
        message: 'Welcome to Express.js!',
        timestamp: new Date().toISOString()
    });
});

// API endpoint
app.get('/api/users', (req, res) => {
    res.json([
        { id: 1, name: 'John Doe', email: 'john@example.com' },
        { id: 2, name: 'Jane Smith', email: 'jane@example.com' }
    ]);
});

// POST endpoint
app.post('/api/users', (req, res) => {
    const { name, email } = req.body;
    
    // Basic validation
    if (!name || !email) {
        return res.status(400).json({ 
            error: 'Name and email are required' 
        });
    }
    
    // Simulate creating user
    const newUser = {
        id: Date.now(),
        name,
        email,
        createdAt: new Date().toISOString()
    };
    
    res.status(201).json({
        message: 'User created successfully',
        user: newUser
    });
});

// Error handling middleware
app.use((err, req, res, next) => {
    console.error(err.stack);
    res.status(500).json({ 
        error: 'Something went wrong!' 
    });
});

// 404 handler
app.use('*', (req, res) => {
    res.status(404).json({ 
        error: 'Route not found' 
    });
});

// Start server
app.listen(PORT, () => {
    console.log(`ğŸš€ Server running on http://localhost:${PORT}`);
});

Installation & Setup

Creating a New Express Project

Project Setup 
# Create project directory
mkdir my-express-app
cd my-express-app

# Initialize npm project
npm init -y

# Install Express
npm install express

# Install development dependencies
npm install -D nodemon

# Create basic structure
mkdir src routes middleware controllers
touch src/app.js src/server.js
package.json Scripts 
{
    "scripts": {
        "start": "node src/server.js",
        "dev": "nodemon src/server.js",
        "test": "echo \"Error: no test specified\" && exit 1"
    }
}

Adding Express to Existing Project

Install Express 
# Install Express in existing project
npm install express

# Add to existing server file
const express = require('express');
const app = express();

// Your existing code here...

Using Express Application Generator

Express Generator 
# Install Express generator globally
npm install -g express-generator

# Create new app with view engine
express --view=ejs my-express-app

# Or create API-only app
express --no-view my-api-app

# Navigate and install dependencies
cd my-express-app
npm install

# Start the application
npm start

Note: The Express generator creates a more opinionated structure with views, public assets, and additional middleware. Great for traditional web applications.
